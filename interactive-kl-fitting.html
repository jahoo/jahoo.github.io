Reading the plots
  • Black curve — the target $p(z)$, a mixture of two Gaussians. Drag the black dots at the mode peaks to move them (horizontal = $\mu$, vertical = $\sigma$).
  • Purple curve — the fitted $q_\phi = \mathcal{N}(\mu, \sigma^2)$. Drag the purple dot at its peak to reposition it (also draggable on the loss landscape).
  • Blue dashed/dotted curves — the exact KL-optimal $q^*$. Dashed = global minimum; dotted = local minima. These recompute when you move the target modes.
  • Rug plot (below x-axis) — the $K$ Monte Carlo particles sampled from $q_\phi$ each gradient step. Each tick marks a sample location $z^{(i)}$. In the forward KL, ticks are purple with opacity $\propto w_i = \tilde{p}(z^{(i)})/q_\phi(z^{(i)})$ (darker = higher importance weight). In the reverse KL, ticks are blue where $r_i > 0$ ($\tilde{p} > q_\phi$) or red where $r_i < 0$ ($q_\phi> \tilde{p}$), with opacity $\propto |r_i|$.
  • Dotted purple curve (forward only) — the theoretical $w(z) = \tilde{p}(z)/q_\phi(z)$. The rug tick opacities are samples of this curve.
  • KL value (top of each panel) — the current KL divergence, computed by numerical integration. The underline color matches the loss landscape at $q_\phi$'s position.
Why MC optimization can get "stuck" where gradient descent wouldn't

None of the methods here compute exact gradients — they all use Monte Carlo estimates based on $K$ samples from $q_\phi$. This means the optimizer can get stuck in regions that are not local minima of the true objective, simply because the gradient estimator has no signal there.

Forward KL is the most striking case: $\mathrm{KL}(p \| q_\phi)$ is convex in the natural parameters of the Gaussian — it has a unique global minimum (the moment-matching solution). True gradient descent would always find it. But with IS, when $q_\phi$ doesn't overlap with a mode of $p$, no samples land near that mode, the IS weights are all $\approx 0$ there, and the gradient estimate is blind to the mode's existence.

Reverse KL has the same issue. Both REINFORCE and reparameterization sample from $q_\phi$, so the gradient can only reflect what $q_\phi$ "sees." If $q_\phi$ is locked onto one mode, samples from the other mode are exponentially rare, and the gradient provides no signal to escape. Note: for the reverse KL, mode-locking is a genuine local minimum of the true objective — even exact gradient descent would stay there. But MC noise can additionally make the optimizer sticky in places that aren't true local minima at all.

You can see this in the rug plot: if all ticks cluster in one region and none reach the other mode, the stochastic gradient can only pull $q_\phi$ toward what it sees. In the forward KL, the ticks near the missing mode will be nearly invisible (low $w$), meaning those samples contribute almost nothing to the gradient.

Try it: set $K$ low (e.g. 16), drag $q_\phi$ onto one mode, and watch — it may stay there indefinitely, despite the loss landscape (for the forward KL) showing no local minimum. Then increase $K$ and watch it escape. This is why SMC / annealing is used in practice: to maintain particle diversity across modes.

How deterministic gradients work

In this mode, gradients are computed exactly (or via high-resolution numerical integration), not estimated from random samples.

Forward KL: The gradient of $\mathrm{KL}(p \| q_\phi)$ has a closed-form solution when $q$ is Gaussian. The optimal $q^*$ is the moment-matching solution: $\mu^* = \mathbb{E}_p[x]$, $\sigma^{*2} = \mathrm{Var}_p(x)$. Since $p$ is a known mixture, these moments are computed directly from the component parameters. The gradient always points toward the unique global minimum, and Adam converges reliably every time.

Reverse KL: The gradient of $\mathrm{KL}(q_\phi \| p)$ requires computing $\mathbb{E}_q[\nabla_x \log p(x)]$. Since both $q$ and $p$ have known closed forms, this expectation is evaluated by numerical integration (trapezoidal rule, ~400 grid points). In 1D this is effectively exact. The reverse KL is nonconvex, so gradient descent finds local minima — but it does so reliably, without the additional failure modes of stochastic estimation.

Why this doesn't scale (and why we need Monte Carlo)

Deterministic gradients work perfectly in this 1D toy setting. In practice, they break down for several reasons:

The curse of dimensionality. Numerical integration on a grid requires $N^d$ evaluations in $d$ dimensions. Here we use ~400 grid points in 1D. A language model has millions of parameters — $400^{1{,}000{,}000}$ grid points is unthinkable. Monte Carlo estimation uses a fixed number of samples $K$ regardless of dimension.

Closed-form moments are rare. The forward KL gradient here exploits the fact that $\mathbb{E}_p[x]$ and $\mathrm{Var}_p(x)$ can be read off the mixture parameters. For complex targets — empirical data distributions, energy-based models — no such closed form exists. In maximum likelihood training, we replace $\mathbb{E}_p[\cdot]$ with a sample average over training data: this is Monte Carlo estimation of the forward KL gradient.

Global evaluation of $p$ is impractical. The trapezoidal rule sums $p(x)$ across the entire domain. In variational inference, $p$ is typically known only up to a normalizing constant, and evaluating it everywhere is infeasible. Monte Carlo methods only need pointwise evaluations at sampled locations.

Try it: switch to Monte Carlo mode to see how stochastic gradient estimation changes the dynamics — particularly how sample-based gradients can be blind to modes that $q_\phi$ doesn't overlap with, even when the true loss landscape has no local minimum there.
